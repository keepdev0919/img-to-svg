## WebP와 SVG, 무엇이 다를까?

WebP는 Google이 개발한 **래스터(비트맵) 이미지 포맷**입니다. JPEG나 PNG보다 파일 크기가 작고 압축 효율이 뛰어나지만, 픽셀로 구성되어 있어 크게 확대하면 깨집니다.

반면 SVG(Scalable Vector Graphics)는 수학적 좌표계로 그린 **벡터 이미지**입니다. 아무리 확대해도 선명하고, CSS로 스타일을 바꾸거나 JavaScript로 애니메이션을 추가하기도 쉽습니다.

### WebP → SVG 변환이 필요한 경우

- 로고나 아이콘을 웹/앱에서 해상도 독립적으로 사용하고 싶을 때
- Figma, Illustrator 등 디자인 툴에 깔끔하게 가져올 때
- 인쇄물을 위해 고해상도 벡터 파일이 필요할 때
- 파일 크기를 최소화하면서 웹 성능을 높이고 싶을 때

---

## WebP vs SVG 비교표

| 항목 | WebP | SVG |
|------|------|-----|
| 파일 형식 | 래스터(비트맵) | 벡터 |
| 확대 시 품질 | 깨짐 | 유지 |
| 파일 크기 | 작음 (손실 압축) | 단순 이미지에서 매우 작음 |
| 브라우저 지원 | 모던 브라우저 전반 | 모든 브라우저 |
| 편집 가능성 | 픽셀 편집 | 코드/툴로 자유롭게 편집 |
| 애니메이션 | 지원 | CSS/JS로 지원 |
| 용도 | 사진, 복잡한 이미지 | 로고, 아이콘, 일러스트 |

### 2단계: WebP 파일 업로드

### 3단계: 변환 옵션 설정 (선택)

## FAQ

### Q1. WebP 파일을 SVG로 변환하면 품질이 좋아지나요?

완전히 새로운 품질이 생기지는 않습니다. 변환기는 원본 픽셀 정보를 기반으로 벡터 패스를 추적(trace)하여 SVG를 만듭니다. 단순하고 명확한 이미지일수록 결과물이 더 깔끔합니다.

### Q2. 사진처럼 복잡한 WebP도 SVG로 변환할 수 있나요?

기술적으로는 가능하지만, 사진처럼 복잡한 이미지는 SVG 파일 크기가 매우 커질 수 있고 결과물이 부자연스러울 수 있습니다. 로고, 아이콘, 일러스트 등 단순한 이미지에 가장 적합합니다.

### Q3. 변환된 SVG를 Figma에서 바로 열 수 있나요?

네. Figma는 SVG 파일을 완벽하게 지원합니다. 변환된 SVG를 Figma에 드래그 앤 드롭하거나 `파일 > 가져오기`로 불러올 수 있습니다.

### Q4. 파일 크기 제한이 있나요?

imgtosvg는 대부분의 일반적인 WebP 파일을 처리할 수 있습니다. 매우 큰 파일의 경우 변환 시간이 더 걸릴 수 있습니다.

### Q5. 변환한 파일은 서버에 저장되나요?

아니요. imgtosvg는 변환 후 파일을 서버에 보관하지 않습니다. 개인 정보와 파일 보안에 안심하고 사용할 수 있습니다.
