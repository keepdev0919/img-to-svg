## 아이콘을 SVG로 변환해야 하는 이유

웹사이트나 앱에 사용하는 아이콘이 PNG나 JPG 형식이라면 확대할 때 픽셀이 깨지는 문제가 생깁니다. SVG(Scalable Vector Graphics)는 수학적 경로로 그려지기 때문에 어떤 해상도에서도 선명하게 표시됩니다. 파일 크기도 일반적으로 PNG보다 훨씬 작아 페이지 로딩 속도 개선에도 도움이 됩니다.

## PNG/JPG 아이콘과 SVG 아이콘 비교

| 항목 | PNG/JPG 아이콘 | SVG 아이콘 |
|------|---------------|-----------|
| 확대 시 품질 | 픽셀 깨짐 | 항상 선명 |
| 파일 크기 | 보통 크다 | 보통 작다 |
| 색상 변경 | 재편집 필요 | CSS로 즉시 변경 가능 |
| 애니메이션 | 불가능 | CSS/JS로 가능 |
| 브라우저 지원 | 전 브라우저 | 모든 최신 브라우저 |
| SEO | 검색 불가 | 텍스트로 인식 가능 |

## 아이콘 SVG 변환 단계별 방법

### 1단계 — 아이콘 이미지 준비

변환 품질을 높이려면 배경이 투명한 PNG 파일이 가장 좋습니다. 배경이 있는 경우 먼저 배경을 제거하거나, 흰색 배경 위의 단색 아이콘이면 그대로 변환해도 됩니다.

### 2단계 — 온라인 변환 도구 사용

[imgtosvg](https://img-to-svg-six.vercel.app)에 접속한 뒤 아이콘 이미지를 드래그 앤 드롭하거나 파일 선택 버튼으로 업로드합니다. 별도의 회원가입 없이 무료로 이용할 수 있습니다.

### 3단계 — 변환 옵션 조정

- **색상 수**: 단색 아이콘이면 2~4색, 그라데이션이 있으면 더 많이 설정
- **노이즈 제거**: 작은 아이콘일수록 노이즈 제거 수치를 높여 깔끔하게 변환
- **경로 단순화**: 복잡한 아이콘은 경로를 단순화해 파일 크기 절감

### 4단계 — 다운로드 및 적용

변환이 완료되면 SVG 파일을 다운로드합니다. HTML에서 `<img>` 태그나 `<svg>` 인라인으로 삽입할 수 있으며, CSS로 색상·크기를 자유롭게 제어할 수 있습니다.

## 아이콘 종류별 변환 팁

### 단색 플랫 아이콘

가장 변환이 쉬운 유형입니다. 색상 수를 2로 설정하면 깔끔한 단색 SVG가 나옵니다.

### 그라데이션·멀티컬러 아이콘

색상 수를 8~16으로 높이고, 변환 후 SVG 코드에서 `linearGradient` 요소를 직접 편집하면 더 정교하게 다듬을 수 있습니다.

### 픽셀 아트 아이콘

픽셀 아트는 벡터화보다 SVG `<rect>` 요소로 픽셀 하나하나를 표현하는 방식이 더 선명합니다. 변환 후 SVG 에디터에서 추가 작업이 필요할 수 있습니다.

## 자주 묻는 질문 (FAQ)

### Q. 변환된 SVG 파일을 피그마에서 편집할 수 있나요?

네. SVG 파일을 피그마에 드래그하면 벡터 레이어로 바로 가져올 수 있습니다. 경로를 선택해 색상·두께를 자유롭게 수정할 수 있습니다.

### Q. 아이콘이 너무 작아서 변환 품질이 낮습니다. 어떻게 하나요?

이미지 편집 도구에서 아이콘을 2~4배 업스케일한 뒤 변환하면 경로 정확도가 높아집니다. 최소 64×64px 이상을 권장합니다.

### Q. SVG 아이콘의 색상을 CSS로 바꾸려면 어떻게 하나요?

SVG를 `<img>` 태그가 아닌 인라인 `<svg>`로 삽입하거나, `<use>` 요소를 활용하면 `fill`, `stroke` 속성을 CSS로 제어할 수 있습니다.

### Q. 변환 후 SVG 파일이 너무 큰 이유는 무엇인가요?

복잡한 경로가 많을수록 파일이 커집니다. SVGO 같은 최적화 도구로 불필요한 메타데이터와 중복 경로를 제거하면 파일 크기를 크게 줄일 수 있습니다.

### Q. 여러 아이콘을 한 번에 변환할 수 있나요?

imgtosvg에서 이미지를 한 장씩 변환할 수 있습니다. 대량 변환이 필요하다면 변환 후 SVG 스프라이트로 합치는 방법을 추천합니다.
