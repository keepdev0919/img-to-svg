## SVG vs PNG — 무엇이 다른가?

이미지를 저장하거나 웹에 업로드할 때 SVG와 PNG 중 어느 포맷을 써야 할지 고민한 적 있으신가요? 두 포맷은 근본적으로 다른 방식으로 이미지를 저장합니다. 올바른 포맷을 선택하면 파일 크기를 줄이고, 화질을 유지하며, 사이트 성능을 높일 수 있습니다.

---

## SVG란?

**SVG(Scalable Vector Graphics)** 는 XML 기반의 벡터 이미지 포맷입니다. 이미지를 픽셀이 아닌 수학적 좌표와 도형으로 저장하기 때문에 **크기를 아무리 키워도 화질이 깨지지 않습니다.**

- 로고, 아이콘, 일러스트레이션에 최적
- 텍스트 에디터로 편집 가능
- CSS·JavaScript로 애니메이션 적용 가능

---

## PNG란?

**PNG(Portable Network Graphics)** 는 픽셀 단위로 이미지를 저장하는 래스터 포맷입니다. **투명 배경(알파 채널)** 을 지원하며 무손실 압축을 제공합니다.

- 사진, 복잡한 그라데이션 이미지에 적합
- 투명 배경이 필요한 UI 요소에 사용
- 확대 시 픽셀이 깨질 수 있음

---

## SVG vs PNG 핵심 비교표

| 항목 | SVG | PNG |
|---|---|---|
| 이미지 방식 | 벡터 (수식 기반) | 래스터 (픽셀 기반) |
| 해상도 독립성 | ✅ 무한 확대 가능 | ❌ 확대 시 품질 저하 |
| 투명 배경 | ✅ 지원 | ✅ 지원 |
| 파일 크기 | 단순 도형은 매우 작음 | 해상도에 비례해 커짐 |
| 애니메이션 | ✅ CSS/JS로 가능 | ❌ 불가 |
| 사진 표현 | ❌ 부적합 | ✅ 적합 |
| 웹 브라우저 지원 | 모든 최신 브라우저 | 모든 브라우저 |
| 편집 용이성 | 코드/벡터 툴로 편집 | 픽셀 편집 필요 |

---

## 어떤 상황에 어떤 포맷을?

### SVG를 쓸 때
- 회사 로고, 아이콘, 버튼 등 **선명도가 중요한 UI 요소**
- 반응형 웹 디자인에서 다양한 화면 크기 대응
- 애니메이션이나 인터랙션이 필요한 그래픽

### PNG를 쓸 때
- 풀컬러 사진이나 복잡한 그라데이션 이미지
- 투명 배경이 필요하지만 벡터화가 어려운 경우
- 레거시 시스템 또는 편집 소프트웨어 호환이 필요할 때

## 자주 묻는 질문 (FAQ)

### Q1. SVG는 모든 이미지에 사용할 수 있나요?
아니요. SVG는 단순한 도형, 로고, 아이콘에 적합합니다. 사진처럼 픽셀 정보가 복잡한 이미지는 PNG나 JPEG가 더 적합합니다.

### Q2. PNG보다 SVG가 항상 파일 크기가 작나요?
단순한 벡터 그래픽은 SVG가 훨씬 작습니다. 하지만 복잡한 도형이 많을수록 SVG의 코드 크기도 늘어나므로 항상 작다고 볼 수는 없습니다.

### Q3. SVG를 PNG로 변환할 수도 있나요?
네. 대부분의 벡터 편집 툴(Inkscape, Illustrator 등)이나 온라인 변환 서비스를 통해 SVG를 원하는 해상도의 PNG로 내보낼 수 있습니다.

### Q4. 웹사이트에서는 SVG와 PNG 중 어느 것이 SEO에 유리한가요?
SVG는 코드가 텍스트 기반이라 검색엔진이 내용을 읽을 수 있고, 파일 크기가 작아 페이지 속도에 유리합니다. 로고나 아이콘은 SVG가 SEO 측면에서 더 유리합니다.
